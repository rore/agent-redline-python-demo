# orders — agent-redline-python-demo

This repo is the Python counterpart to [`agent-redline-demo`](https://github.com/rore/agent-redline-demo). Same purpose: exercise [`agent-redline`](https://github.com/rore/agent-redline) end-to-end through the three canonical PR states (BLUE / RED-checkpoint / BOUNDARY_VIOLATION).

The service itself is a deliberately tiny hexagonal FastAPI app — just enough structure for agent-redline's Python extension (using `import-linter`) to enforce a layered architecture.

## What's here

| Path | What it is |
|---|---|
| `src/orders/` | The service: `api/`, `application/`, `domain/` (with `repositories/` ports), `infrastructure/` |
| `pyproject.toml` | Includes `[tool.importlinter]` contracts |
| `agent-policy.yaml` | Generated by agent-redline bootstrap; declares zones, boundary contracts, the `architecture-review` checkpoint |
| `scripts/agent-redline-check.sh` | Local pre-push: runs `pytest`, `lint-imports`, then the agent-redline reporter |
| `scripts/run-import-linter.py` | Adapter: runs `import-linter` and emits `boundary-violations.json` |
| `scripts/agent-redline-report.py` | Vendored reporter (auto-synced from `rore/agent-redline`) |
| `.github/workflows/agent-redline.yml` | CI: boundary job + reporter job |
| `docs/agent/` | Per-checkpoint reference docs |

## Three canonical PRs

| Branch | Verdict |
|---|---|
| `demo/blue-only-pr` | BLUE; CI green; no checkpoint |
| `demo/red-with-checkpoint-pr` | RED; CI green via `architecture-reviewed` label |
| `demo/boundary-violation-pr` | BOUNDARY_VIOLATION; CI red; cannot merge |

## Run locally

```bash
pip install -e '.[dev]'
pytest
bash scripts/agent-redline-check.sh
```
